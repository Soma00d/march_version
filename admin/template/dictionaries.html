Global Information
Name:
Designation of the component according to the new names standard.
Photo Link :
Allows to associate a photo with a component in the interface.
Is Safety:
Indicates if the component has the safety characteristics (safety_loop_freq signal)
Is Enable:
Indicates if the component has the enable characteristics (tsui_enable signal and HW validation for joystick types)
Is CDRH:
Indicates if the component has the property CDRH.
Reference TST:
Designation of the component according to the generalized names/references.
Description:
Description of the component.
Area Engineering:
In engineering mode, allows to allocate the components in different areas over the user interface.
Is Final:
Indicates if the component shall be tested during Final Test process in Repair mode.
Timer (seconds):
Indicates the time during which the operator can test a component while executing a final test in Repair Mode.
CAN Information
CAN ID:
Is the CAN identifier of the component. 11 bits for Elegance, 29 bits for Omega. Here, the CAN ID shall not consider the node ID.
CAN Data Press:
Is the CAN message sent by the TSUI when a button is pressed.
CAN Data Release:
Is the CAN message sent by the TSUI when a button is released.
LED Information
Is Led:
Indicates if the button has a backlight/led to be tested.
On Signal:
Is the CAN message sent to the TSUI to switch ON a led/backlight/display/buzzer.
OFF Signal:
Is the CAN message sent to the TSUI to switch OFF a led/backlight/display/buzzer.
DIM Signal:
Is the CAN message sent to the TSUI to DIM a led/backlight/display.
FLASH Signal:
Is the CAN message sent to the TSUI to BLINK a led/backlight/display.

Global Information
Name:
Designation of the component according to the new names standard.
Photo Link :
Allows to associate a photo with a component in the interface.
Is CDRH:
Indicates if the component has the property CDRH.
Reference TST:
Designation of the component according to the generalized names/references.
Description:
Description of the component.
Is Final:
Indicates if the component shall be tested during Final Test process in Repair mode.
Timer (seconds):
Indicates the time during which the operator can test a component while executing a final test in Repair Mode.
CAN Information
CAN ID:
Is the CAN identifier of the component. 11 bits for Elegance, 29 bits for Omega. Here, the CAN ID shall not consider the node ID.
X Position:
Indicates the bits associated to the CAN data frame sent by the TSUI for the x-axis movement of the joystick.
Y Position:
Indicates the bits associated to the CAN data frame sent by the TSUI for the y-axis movement of the joystick.
Calibration Subindex X:
Indicates the sub-index for SDO CAN protocol used for X axis calibration of the joysticks.
Calibration Subindex Y:
Indicates the sub-index for SDO CAN protocol used for Y axis calibration of the joysticks.
Threshold Min Axis:
Indicates the threshold value allowed for minimum axis X/Y raw position calibration.
Threshold Max Axis:
Indicates the threshold value allowed for maximum axis X/Y raw position calibration.
Threshold Min Zero
Indicates the minimum threshold value allowed for zero position calibration.
Threshold Max Zero
Indicates the maximal threshold value allowed for zero position calibration.
