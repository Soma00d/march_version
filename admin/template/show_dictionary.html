CAN Information
CAN ID:
Is the CAN identifier of the component. 11 bits for Elegance, 29 bits for Omega. Here, the CAN ID shall not consider the node ID. It is a 4 bytes hexadecimal value, you shall add zeros if necessary.
CAN Data Press:
Is the CAN message sent by the TSUI when a button is pressed.
CAN Data Release:
Is the CAN message sent by the TSUI when a button is released.
LED Information
Is Led:
Indicates if the button has a backlight/led to be tested. The backlight for Emergency Stop button works differently, please select Emergency Stop LED for this one.
ON Signal:
Is the CAN message sent to the TSUI to switch on a led/backlight/display/buzzer. It is composed of: CAN-ID (4 bytes, i.e. 00000428h) and CAN-DATA (8 bytes, i.e. 0200000000000000h). For a 7-segments display, one byte is the value of the character to be displayed.
OFF Signal:
Is the CAN message sent to the TSUI to switch off a led/backlight/display/buzzer. It is composed of: CAN-ID (4 bytes, i.e. 00000428h) and CAN-DATA (8 bytes, i.e. 0000000000000000h).
DIM Signal:
Is the CAN message sent to the TSUI to dim a led/backlight/display. It is composed of: CAN-ID (4 bytes, i.e. 00000428h) and CAN-DATA (8 bytes, i.e. 0100000000000000h). For a 7-segments display, one byte is the value of the character to be displayed.
FLASH Signal:
Is the CAN message sent to the TSUI to blink a led/backlight/display. It is composed of: CAN-ID (4 bytes, i.e. 00000428h) and CAN-DATA (8 bytes, i.e. 0300000000000000h). For a 7-segments display, one byte is the value of the character to be displayed.

Name:
Designation of the component according to the new names standard.
Photo Link :
Allows to associate a photo with a component in the interface. Format: [folder_name]/[file_name].[ext]
Is CDRH:
Indicates if the component has the property CDRH.
Type:
Indicates if the component is a joystick or a mushroom.
Reference TST:
Designation of the component according to the generalized names/references.
Description:
Description of the component.
Is Final:
Indicates if the component shall be tested during Final Test process in Repair mode.
Timer (seconds):
Indicates the time during which the operator can test a component while executing a final test in Repair Mode.
CAN Information
CAN ID:
Indicates if the button has a backlight/led to be tested. The backlight for Emergency Stop button works differently, please select Emergency Stop LED for this one.
X Position:
Indicates the bits associated to the CAN data frame sent by the TSUI for the x-axis movement of the joystick.
Y Position:
Indicates the bits associated to the CAN data frame sent by the TSUI for the y-axis movement of the joystick.
Calibration Subindex X:
Indicates the sub-index for SDO CAN protocol used for X axis calibration of the joysticks. Please refer to document “joystick calibration protocol” to get the correlation between sub-index and joystick axis.
Calibration Subindex Y:
Indicates the sub-index for SDO CAN protocol used for Y axis calibration of the joysticks. Please refer to document “joystick calibration protocol” to get the correlation between sub-index and joystick axis.
Threshold Min Axis:
Indicates the threshold value allowed for minimum axis X/Y raw position calibration.
Threshold Max Axis:
Indicates the threshold value allowed for maximum axis X/Y raw position calibration.
Threshold Min Zero
Indicates the minimum threshold value allowed for zero position calibration.
Threshold Max Zero
Indicates the maximal threshold value allowed for zero position calibration.
